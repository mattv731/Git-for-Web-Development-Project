    37. Walk me through Lambda's git flow. 
1. Fork the repo to make your own copy.
2. Clone your repo. This pulls down the files so you can work on them on your laptop.
3. Open git bash and navigate to where you would like to keep the files. Then type "git clone" and paste the url(clone)
you copied.
4. Go into the new repository by typing cd and the name of the repo.
5. Create a branch by typing "git checkout -b branch-name" branch name is your first and last name.
6. Open the project up in VS code.
7. Work on the project.
8. Push the changes you made by doing the following.
	a. "git add ."
	b. "git commit -m 'message goes in here'"
	c. "git push -u origin branch-name"
9. Go back to github and create a pull request.
10. Click on the button in github that says "compare and pull request" this will take you to a new screen.
11. Open the drop down menu and choose your repo base(my name).
12. Ensure that you are comparing 'master' or 'main' to my branch then click pull request.
